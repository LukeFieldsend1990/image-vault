02

Security & Custody

The moat: control, not just storage
AES‑256
Encryption at rest
Scan data encrypted at rest on Cloudflare R2; TLS 1.3 in transit.
2×
Dual-custody download
Both licensee and talent/rep must clear 2FA before a token issues. No single party releases a likeness.
SHA‑256
Integrity hashing
Every scan file & token streamed-hashed for tamper detection.
AuthenticationRFC 6238 TOTP 2FA, JWT sessions, 7-day refresh tokens. Admin actions require a fresh per-action TOTP code (dual-custody analogue).
Download tokensTime-limited, single-purpose tokens — never standing access.
Admin API tokensExpire in ≤90 days, always revocable, SHA-256 hashed (never stored in plaintext). Rate-limited to 120 calls/min; 5 TOTP attempts / 5 min.
Audit loggingEvery privileged action logged with secrets redacted. Security & download events captured with IP, device & timestamp.
Compliance ledgerSAG-AFTRA Article 39 obligations tracked in an append-only, SHA-256 hash-chained ledger — court-admissible chain of custody.
Anti-piracyGeometry fingerprinting & watermarking (128-bit) on assets flowing into render engines via the controlled Render Bridge.

Security model is access-control based — server-mediated, with storage-provider encryption at rest. It is deliberately not marketed as "zero-knowledge" or "end-to-end encrypted."
